Создание пользователя и установка пароля(Аутентификация)

sudo adduser [имя] - создаёт нового пользователя. Там скипаем всё (жмём enter) кроме пароля
sudo passwd [имя] - проверка на сущ-ние
sudo usermod -aG sudo testuser - для прав админа новому пользователю 
groups testuser - проверка на наличие в группе

sudo apt install git
git clone ссылка https://github.com/имя_пользователя

Изменение пароля пользователя (Аутентификация)

passwd - Измени пароль текущего пользователя
sudo passwd [имя] - Измени пароль другого пользователя (требуются права sudo)


Отключение входа пользователя (Аутентификация)

sudo passwd -l testuser - Заблокировать пользователя (запретить вход)
sudo passwd -u testuser - Разблокировать пользователя


Настройка прав доступа к файлу/каталогу (Авторизация)

touch [имя].txt - Создай тестовый файл
sudo chown testuser:testuser [имя].txt - Измени владельца файла (теперь [имя] является владельцем файла и группы)
chmode 600 [имя].txt - Изменить права доступа к файлу.(Только владелец может читать и писать, никто другой не имеет доступа). 600 - это права. 
Первая цифра - права владельца testuser, вторая - права группы testuser, третья - права всех остальных
4 - чтение (read)
2 - запись (write)
1 - выполнение (execute)
Сумма этих чисел определяет права. Например, 6 (4+2) означает чтение и запись.

SSH ключи (Аутентификация)

ssh-keygen -t rsa - Сгенерируй пару ключей для пользователя (предположим, ты залогинен под testuser):

Следуй инструкциям, можно просто нажимать Enter для значений по умолчанию.
ssh-copy-id testuser@localhost - Скопируй публичный ключ на сервер (или в домашнюю директорию testuser):
Введи пароль testuser.
Теперь ты можешь заходить по SSH без пароля:
ssh testuser@localhost
